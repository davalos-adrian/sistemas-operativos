Integración de clientes en un dominio

Es muy importante que cuando se quiera agregar un cliente al dominio se verifique una serie de requisitos:

  1. El quipo tiene un nombre único en la red
  2. El equipo tiene una direción IP dentro del mismo rango de direcciones IP que el servidor
  3. La máscara de red tiene que ser la misma que la del servidor u otra que permita verla en la red
  4. El grupo de trabajo al que pertenece el equipo no es importante hasta este punto
  5. Introducir la direccion IP del servidor como DNS princial del equipo
  6. Comprobar y verificar que el cliente tiene conectividad con otros equipos utilizando cmd. Habrá que hacer un ping a la IP del servidor
  7. Asegurarnos de que el usuario Administrador del cliente tiene contraseña
  8. Haber iniciado sesión en el equipo localmente con la cuenta local del administrador

Una vez que se cumplan todos los requisitos, ya se tiene un equipo listo para ser integrado en el dominio. Para realizar esta operación hay que serguir lo siguientes pasos:

  1. Desde el equipo, se tiene que ir a propiedades del equipos o sistema, en la pestaña de Nombre del equipo
  2. Pulsar el botón Cambiar. Se verá una pantalla como la siguiente imagen. Ahí se introducirá el nombre del dominio al que se quiere unir

Aparecerá una pantalla para iniciar sesión con un usuario que se haya dado de alto en el dominio. Hechos estos pasos, la siguiente vez que el cliente quiera iniciar sesión, aparecerá el nombre del dominio seguido del nombre del usuario como en la siguiente imagen. Esto quiere decir que el usuario y ese equipo ya se encuentran en el dominio

Conceptos de permisos y derechos, recursos compartidos

En un dominio se pueden compartir recursos asignandose permisos de acceso solamente a usuarios globales del dominio, pero desde los clientes se pueden compartir recursos para el resto de usuarios de la red y también asignar permiso de acceso a usuarios locales Para compartir una carpeta es necesario seguir los siguientes pasos. Dentro de herramiuentas administrativas > administración de equipos > recursos compartidos, se pueden ver los recursos que el propio servidor ha comaprtido automaticamente al crearse el controlador de dominio

Ahora sí, para propósitos de este ejemplo, se van a crear 3 carpetas bases de datos, plantillas word y copias de seguridad Para compartir una carpeta primero se tiene que activar el uso compartido de carpetas, hay que ir a centro de redes y recursos compartidos, después hacer click en cambiar configuración de uso compartido avanzado, en los botones que se muestran, seleccionar Activar uso compartido de archivos y Activar la detección de redes, pulsar guardar. Después, es tan sencilo como hacer click derecho en el recurso que queremos compartir, y seleccionar compartir se mostrará un asistente donde preguntará por cuales usuarios y grupos deben de poder acceder a ese recurso.

Tipos de permiso de acceso a recursos

Existen varios tipos de permisos para poder acceder a los recursos, los cuales se enlistan en la tabla siguiente:

Tipo de permiso Tipo de acceso permitido Version de windows server
Cambio Los usuarios pueden cambiar el contenido de archivos, pero no pueden crear archivos o carpetas 2003
Permitir/denegar Permite o no los permisos anteriores al usuario o grupo seleccionado 2003/2008/2008R2
Lectura Los usuarios pueden ver el contenido de una carpeta y abrir archivos y carpetas 2003/2008/2008R2

Conclusión

Uno de los aspectos más importantes cuando se tienen muchos usuarios y recursos es el tipo de acceso que se le otorga acada usuario sobre cierto recursos, ya que si todos los usuarios tuvieran control total no tendría sentido que existiera un controlador de dominio en esta parte, puesto que, a la hora en que se compartiera un archivo todos los usuarios tendrían acceso total a este recurso y se perdería el control sobre el mismo, cualquier usuario que no tenga conciencia de lo que está haciendo en un recursos podría modificarlo o incluso peor, borrarlo. Es por esto que la otorgación de permisos sobre un recurso es una parte vital en cualquier sistema operativo, ya sea en red o no, siempre que se cuente con acceso a multiples usuarios.
