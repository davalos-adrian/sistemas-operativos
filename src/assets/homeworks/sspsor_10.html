Introducción y características de los sistemas operativos en red

Linux Ubuntu viene normalmente disponible en 2 distribuciones, desktop y server. Para esta ocasión utilizaremos ubuntu server. Ubuntu server nos permitirá utiliza un equipo como servidor, este gestionará inicios de sesión de equipos clientes. Los recursos a los que los clientes tienen acceso se gestionan de forma centralizada desde este equipo servidor. Servidores y clientes quedan coordinados para que todos los elemetnos de la red funcionen correctamente.

Es cierto que la versión de escritorio de ubuntu puede utilizarse indistintamente como servidor, pero todos los servicios necesarios para fungir como uno, tendrán que ser descargados manualmente. Y es así como la distribución de ubuntu server viene destinada exclusivamente para esta funcionalidad, ser un servidor de red, por lo que durante la instalación, permitirá configurar los servicios necesarios para que el equipo se convierta en uno.

Windows server vs Linux server

Ventajas de windows server

Todas las aplicaciones de windows como office, outlook, sql server al ser propietarias de Microsoft, se integrarán a tus servidores windows al instante. Si piensa utilizar servicios de windows y software, resulta lógico instalar un servidor nativo como windows server. Windows server se considera como una solución completa que es muy fácil de instalar y configurar. Si el equipo de trabajo trabaja bajo frameworks como ASP y ASP.Net, estos servicios solo funcionarán en windows server.

Ventajas de linux server

Una de las mayores ventajas de linux es que es de código abierto, por lo tanto está abierto a modificaciones para mejorar o arreglas características del sistema operativo. Otra de las mayores ventajas es que es totalmente gratuito. Linux tiene compatibilidad con otros softwares de código abierto y provee una rápida interfaz. Generalmente, los servidores de linux utilizan muchos menos recursos, ya que están diseñados para correr solamente las funciones necesarias. Linux es más seguro que windows. La mayoría de los malware están diseñados para dañar sistemas windows.

En resumen
  1. Es más dificil configurar y manejar un servidor linux, para hacer esto en windows no se necesita ser un experto
  2. Linux es una opción mejor para desarrolladores que quieran montar un servidor Apache o NGINX. o incluso desarrolladores que quieran trabajar con bases de datos MySQL
  3. El soporte técnico de windows viene incluido, además de soporte para actualizaciones y errores de seguridad, en linux es necesario pagar una cuota si se necesita un técnico.
  4. A la larga, es más costo mantener los servidores

Planificar una instalación de ubuntu server

Primeramente, hay que plantearse una serie de preguntas.
  • ¿Cuál es el uso planeado de este sistema y quién se lo va a dar?
  • ¿Existen perspectivas de que esto cambie a corto o medio plazo? ¿Y a largo?
  • ¿De qué rescursos se dispone para lograr este fin?
  • ¿Para qué y por qué necesito un sistema informático?
  • ¿Cómo espero que me ayude el sistema?
  • ¿Cuáles son los recursos de los que dispongo?

Una vez contestadas estas preguntas, y concluyendo que es necesario realizar una instalación de algún servidor, es perfectamente posible continuar. Para la siguiente y última práctica, será necesario instalar un servidor de linux, y esto será parte de la planeación. Como no se dará acceso a probablemente más de 2 usuarios (computadoras con las que haré pruebas) no será necesario contar con muchos recursos, por lo que quedará de la siguiente manera.

  1. Se realizará la instalación de linux ubuntu server 18.04, por razones de compatibilidad
  2. Se instalará en una laptop dentro de una partición de no más de 60 GB
  3. Se configurará esa laptop como controlador de dominio
  4. Se crearán 2 cuentas de usuario para poder acceder al dominio desde otro equipo
  5. Se instalará OpenSSG Server para poder utilizar SSH al servidor
  6. Se instalará un servicio DNS
  7. Se instalará un servicio LAMP, para poder contar con un servidor web y hospedar páginas web
  8. Se instalará Samba, para poder compartir archivos dentro del dominio
  9. Se instalará un servidor FTP para tener acceso al sistema de archivos del sistema

    10. Conclusión

    Es muy importante que se haga la planificación de la instalación correctamente conforme a las necesidades del sistema que se esté planeando hacer. Igualmente hay que planear para saber si los recursos con los que contamos pueden mantenerse de esa manera o no, que generalmente esto no sucede, puesto que en cualquier empresa (de ser el caso) el tamaño de la información y los usuarios siempre va en constante crecimiento, y de la mano los recursos del sistema.
