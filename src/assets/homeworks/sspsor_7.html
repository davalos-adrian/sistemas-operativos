Gestión de grupos de usuario y creación de usuarios

Para crear un grupo se debe tener clara la infraestructura de la empresa, los usuarios, grupos y usuarios que pertenecerán a cada grupo. Es necesario tener en cuenta que el nombre del grupo debe ser único dentro de todo el dominio

Pasos para crear un grupo

Primero hay que desplegar la unidad organizativa que contiene grupos para poder acceder a la creación de estas mismas.

Después hay que hacer click derecho, seleccionar Nuevo y Grupo, se desplegará un cuadro de dialogo para configurar el nuevo grupo.

En el cuadro de dialogo habrá que configurar el nombre del grupo para versiones nuevas de windows server y para grupos anteriores a windows 2000. También habrá que configurar el ambito y el tipo del grupo

En este ejemplo se creó un grupo llamado RH de tipo grupo de seguridad, existen 2 tipos de grupos como se explicó

  1. Grupos de seguridad: Son creados para dar o eliminar permisos sobre los recursos del dominio
  2. Grupos de distribución: Son creados para realizar instalaciones remotas de software en equipos clientes en los qeu validan usuarios que pertenezcan al grupo

Agregar usuarios un grupo del dominio

Para este ejemplo se va a crear una unidad organizativa llamada departamentos, dentro de esta unidad se creará un grupo local llamado contabilidad, otro grupo global llamado almacen y otro grupo universal llamado mantenimiento.

Hay que seleccionar los usuarios que se quieran agregar al grupo, dar click derecho y seleccionar agregar al grupo.

Se escriben los nombres de grupos que se quieran añadir y dar click en comprobar nobmres.

También es posible agregar usuarios a los grupos desde las propiedades del grupo seleccionando la pestaña de miembros y dando click en agregar.

Creación de usuarios en el dominio

Para crear un usuario se tiene la unidad organizativa Users o usuarios, dentro de esta si se da click derecho en nuevo y luego en usuario, aparecerá una ventana para configurar los datos del usuario.

Conclusión

Una vez más observamos el control que se tiene sobre los usuarios del dominio, de aquí se lleva el nombre de controlador de dominio, tiene control sobre todas las unidades y objetos del dominio, incluyendo grupos, usuarios, unidades organizativas, sobre todo.
