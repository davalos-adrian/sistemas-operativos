Arranque del sistema

Todos los sistemas operativos necesitan correr por un proceso de arranque para poder comenzar a funcionar. Durante este proceso, el sistema operativo se prepara para ser utilizado por el/los usuario/s. Generalmente, el proceso de arranque del sistema cuenta de 2 fases:

  • Arranque del hardware
  • Arranque del S.O.

Todo este proceso inicia desde que se carga el programa de arranque disponible en la memoria ROM. El programa inicializador de la ROM se encarga de comprobar el sistema, detectando características y comprobando su funcionamiento, también lee y almacena el programa que carga el S.O. y procede a pasarle el control a este. Este programa se encuentra en los sectores conocidos como MBR y se encarga de cargar el núcleo del S.O. y pasarle el control.

El núcleo continúa con el arranque realizando una comprobación del hardware del sistema. Después de inicializar suis tablas internas y generar las estructuras de datos necesarias, crea el proceso Init y le cede el control. Este proceso termina el proceso de arranque, dejando listo el sistema en modo multiusuario para que sea utilizado. Este proceso verifica los sistemas de archivos, monta los sistemas de archivos permanentes, activa las áreas de memoria de intercambio o swap, activa los demonios (servicios en windows) y finalmente habilita el inicio de sesión a los usuarios del sistema.

Parada del sistema

Es posible detener un sistema operativo, puesto que en ocasiones será necesario un reinicio o apagar por completo el sistema. Este proceso sigue los siguientes pasos:

  1. Se notifica a los usuarios
  2. Se envía una señal de finalización a los procesos
  3. Se detienen los servicios o demonios
  4. Se "kickea" a los usuarios que estén utilizando el sistema
  5. Procesos que quedaron activos se les manda la señal de "kill"
  6. Se actualizan o se ejecutan las actualizaciones de disco pendientes
  7. Se ejecuta la orden de apagado (apagar, reinicio, etc.)

Causas de arranque y parada de sistemas ajenas al usuario

No siempre será el usuario quien mande la señal de finalización del sistema y no siempre se iniciará el sistema cuando el usuario lo finalice, puesto que existen causasn externas al usuario que pueden evitar el inicio del sistema o que pueden causar una finalización imprevista del sistema. Estas se enlistan a continuación (recordar que son algunas, ya que las razones pueden ser muchas).

Causas de finalización del sistema

  • Fallos en el hardware
  • Errores en el hardware irrecuperables
  • Fallos en la corriente electrica
  • Problemas en la entrada o salida
  • Problemas en algún sistema de archivos

Estos fallos en la paltaforma de windows, generalmente son representados mediante una "pantalla azul", indicando el tipo de error y algún código para acudir al soporte técnico, como se muestra en la siguiente figura.

Existe un equivalente en linux llamado panico de kernel, es una pantalla MUY POCO COMÚN, puesto que los sistemas linux son muy buernos manejando hardware. Generalmente estos errores se producirían cuando se cuenta con hardware dañado.

Problemas al iniciar el sistema

  • Hardware dañado o con errores
  • Errores al leer el sistema de archivos
  • Sectores dañados o que no perteneces al sistema de archivos
  • Hardware incompatible
  • Errores en la configuración del sistema

Servicios de directorio y dominio

El servicio de directorio es una aplicación o un conjunto de aplicaciones que almacena y organiza la información sobre los usuarios de una red de ordenadores y sobre los recursos de red que permite a los administradores gestionar el acceso de usuarios a los recursos sobre dicha red. Además, los servicios de directorio actúan como una capa de abstracción entre los usuarios y los recursos compartidos.

  • Active directory (Windows)
  • Open directory (Mac OS X)
  • NIS Network information system (UNIX)
  • Servidor de directorio de Red Hat
  • Servidor de directorio de Apache
  • Servidor de Sun Java System
  • Entre otros.

Los administradores de redes tienen claro que la forma más eficiente de aprovechar el potencial de los equipos es mediante la creación de dominios. Los dominios almacenan información administrativa de manera centralizada, como son las cuentas de usuario, impresoras, directorios, etc. facilitando así las labores del administrador. Específicamente en windows, cuando se instala un directorio activo en un ordenador, este se convierte en un servidor, o más correctamente, en un controlador de dominio.

Conclusión

Los controladores de dominio y los servicios de directorio son una parte fundamental de los sistemas operativos en red, es la columna de estos, puesto que estos proporcionan el control y la administración de todos los equipos que forman parte del dominio, brindan los servicios necesarios para organizar y almacenar la información de los usarios dentro de la red. Es importante tenerlos en cuenta durante el transcurso de toda la materia, pues serán una parte primordial durante la duración de esta.
